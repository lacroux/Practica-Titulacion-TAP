
Sistema de Gestión Web y de Escritorio - Control Operativo Centralizado 

🚀📋 Descripción del Proyecto

Este proyecto consiste en el diseño e implementación de un sistema de gestión centralizado desarrollado 
para automatizar y optimizar procesos clave como el control de entradas, asignación de tareas, registro 
de mermas y generación de informes detallados. La solución busca eliminar la gestión manual y la falta 
de trazabilidad, aumentando la eficiencia operativa y facilitando la toma de decisiones informadas.

🛠️ Stack Tecnológico
Para garantizar la escalabilidad y seguridad de los datos, se utilizó el siguiente conjunto de tecnologías: 

- Control de Versiones: Git con GitHub.
- Gestión de Proyectos: Trello y OpenProject.

📑 Análisis de Requerimientos
Requerimientos Funcionales (RF) Destacados

El sistema cuenta con una arquitectura basada en roles y permisos 
para garantizar un funcionamiento controlado.

- RF 01 - Autenticación: Acceso restringido mediante correo y contraseña según tipo de rol.
- RF 03 - Gestión de Tareas: Módulo para visualizar, crear, editar y eliminar asignaciones.
- RF 06 - Registro de Mermas: Registro de productos perdidos y causas para el análisis de costos operativos.
- RF 07 - Generación de Informes: Reportes personalizados exportables en formatos PDF y Excel.

Requerimientos No Funcionales (RNF)

- Capacidad: Soporte para hasta 250 usuarios con sesiones concurrentes.
- Rendimiento: Tiempo de respuesta inferior a 3 segundos para consultas simples.
- Interfaz: Diseño intuitivo y responsivo para visualización en diversos dispositivos.

⚙️ Metodología ScrumEl desarrollo se llevó a cabo utilizando la metodología ágil Scrum, permitiendo entregas
incrementales y ajustes dinámicos según las necesidades de la empresa. El proyecto se dividió en 5 etapas clave:

- Sprint 1: Creación de Base de Datos y servicios básicos de backend.
- Sprint 2: Desarrollo de servicios avanzados y frontend inicial (Login/Menú).
- Sprint 3: Implementación de formularios específicos y listados dinámicos.
- Sprint 4: Finalización del desarrollo, documentación técnica y capacitación.
- Sprint 5: Pruebas de aceptación, despliegue y soporte post-implementación.
